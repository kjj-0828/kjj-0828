The usage of vue js framework for web application creation

使用vue js框架创建web应用程序

本文着重介绍了前端框架的优点。它创建了一个小网页，展示了Vue JS JavaScript前端框架的一些基本功能及其实现的易用性。它说明了为什么在开发中需要框架，以及公司如何利用它们为自己谋利。由于它们可以被使用并加速开发过程，因此可以在学校学习它们，以便更容易地掌握开发概念。

用户的增长完全改变了我们的日常生活。我们大多数人使用谷歌来寻找几乎所有我们需要的信息，我们去Facebook和我们的朋友聊天或组织活动，我们使用Instagram分享我们的照片和日常故事等。互联网上的数据量呈指数级增长。

这些变化也在全球经济中掀起了一股浪潮。不同的公司不得不调整自己以适应这种变化。他们建立了只处理信息技术的部门。

对于大多数公司来说，区别在于他们拥有的开发人员需求。并非所有开发者都拥有相同的技能，因此你需要找到那些能够创建你所需要的网站或网页应用的人。为了达到本文的目的，我们将只使用web应用程序。其原因是在任何类型的组织中广泛使用，例如电子商务。接下来，我们检查用于创建应用程序的最常用框架

公司通常将收入的4%到6%用于IT部门(Morley, 2015)。这种巨额的现金分配部分源于软件开发所花费的长时间。创建一个专业的web应用程序包含许多重复的活动，对软件开发人员有巨大的需求。这些重复的活动可以被抽象到一个框架中，因此为了更容易地创建网站和web应用，我们创建了不同的web开发框架。

Vue JS到底是什么?我们可以在官方文档中找到答案:“Vue是一个用于构建用户界面的渐进框架。与其他单片框架不同的是，Vue从头开始设计，逐步被采用。核心库只关注视图层，很容易获取并与其他库或现有项目集成。”

Vue Js带有反应系统(You, 2019)。第一步是创建一个Vue实例。在Vue的实例化过程中，您创建了一些属性。当您定义数据属性(Vue对象的特定属性)时，Vue会为其创建getter和setter方法。这些是绑定到JavaScript对象的JavaScript函数，其目的是动态返回计算值，并动态查找指定属性的更改。

总之，我们可以说，在Vue JS框架中工作是简单易懂的，即使对于没有经验的开发人员也是如此。创建一个基本应用程序所需的知识库并不高，因此您可以非常快速地学习基础知识。它很容易在课堂或学校使用，使学习web开发更容易和更具互动性的体验。即使我们不打算在信息技术领域工作，我们也可以通过它学习web开发的基础知识，并在浏览过程中更加意识到在幕后发生了什么。

    根据经验，MVC和MVVM都有各自的优缺点，选择其中一种作为设计模式在很大程度上取决于项目架构。

    (1)模型对象保存和管理应用程序的数据，

    (2)视图对象呈现交互的可视化元素，并将用户操作发送给控制器对象。

    (3)控制器对象绑定模型对象和视图对象，并包含应用程序逻辑[15]

    事实上，MVC和MVVM都坚持关注点分离(SoC)的概念，这提高了代码质量[9]。然而，MVVM比MVC分离了更多的关注点，MVC降低了复杂性，最大化了可测试性和可重用性。MVVM引入了一个名为ViewModel的新对象。该对象具有为View对象准备数据的逻辑，这是MVC[5]中的Controller对象的职责.此外，类似于MVC视图对象和控制器对象是成对工作的，但MVVM将它们合并成一个对象，并将其称为视图对象[10]。因此，MVVM增加了数据独立和应用逻辑封装[9]。

根据用户请求，服务器将所有请求信息发送给特定的控制器，该控制器处理来自客户端的整个请求，并告诉服务器的其余部分如何处理该请求，它充当其他两个部分(模型和视图)之前的中间人不应该包含太多代码，当控制器收到请求时的第一件事是根据其请求向模型请求消息

模型负责处理请求的所有数据逻辑，这意味着模型与数据库交互并处理数据的所有验证 保存 更新 删除等，控制器不应该直接与数据逻辑交互，而应该只使用模型来执行这些交互，这意味着控制器永远不必担心如何处理它发送和接受的数据，而只需要告诉模型该做什么，并根据模型返回的内容进行响应，这也意味着模型永远不必担心处理用户请求以及在失败或者成功时该做什么，所有这些都由他们的控制器处理，模型只关心与数据的交互，在模型将响应发送回控制器后，控制器随后需要玉视图交互，将数据呈现给用户

视图只关心如何显示控制器发送给他的消息，这意味着视图是一个将控制器发送的数据动态呈现HTML的模板文件 视图不关心处理数据的最终表示 而只关心如何表示数据

视图将把它最终的表示发送回控制器 控制器将处理表示发送回用户

关于这个设计最重要的一点是，视图和模型从不相互作用，模型和视图间的交互勾兑通过控制器完成的

这意味着数据的表示和数据的逻辑是完全分离的，这使得创建复杂的应用程序更加容易
