  5. 组件设计有什么原则呢？

    答：这个我没有考虑这个方面的

    【补充】

    高层模块不依赖底层模块，即为依赖倒转原则。
    内部修改关闭，外部扩展开放，即开闭原则。
    聚合单一功能，即为单一职责原则。
    低知识要求，对外接口简单，即迪米特法则。
    耦合多个接口，不如独立拆分，即接口隔离原则。
    合成复用，尽量不使用继承，即合成复用原则。

  6. 你学习过一些设计模式吗？

    答：我记得有单例模式，还有跟独立性有关的一个

    【补充】

    （1）外观模式，客户端—–>facade(外观)—>subSystem(子系统)

    外观模式将子系统复杂的业务逻辑进行抽象，提供一个更统一、简洁、更易用的API，比如，JQuery就是把负责的DOM操作进行抽象和封装，并且兼容各个浏览器

    （2）代理模式， Real Subject(真实对象)、proxy（代理对象）、subject（接口 2个对象都要实现的接口）

    当需要花费大量的时间去访问对象，或者访问对象需要增加额外的逻辑，考虑使用代理模式

    （3）工厂模式， 原材料—->汽车服务中心—–>各大制造工厂——>各大汽车

    工厂就是创建出其他对象的对象，根据传入的参数不同，生产的对象不同，当构造函数过多，创建的对象存在某种关系(同一个父级，实现同一个接口)可以考虑工厂模式

    （4）单例模式， 一个类只能有一个实例，提供一个访问他的全局访问点

    当需要一个类去贯穿整个系统的时候，考虑单例模式，比如：弹窗，无论点击多少次，弹窗应该只创建一次

    （5）策略模式

    针对同一个操作，在不同的场景下，有不同的策略算法，比如登录鉴权，针对不同的设备方式，手机，邮箱，第三方有不同的策略算法

    （6）迭代器模式 ，遍历容器并且访问容器的元素，不管容器的数据结构是什么

    es6提供了迭代器Iterator，例如for of 可以遍历一个数组

    （7）发布订阅模式， （观察者模式） 被观察者、观察者

    被观察者维护一组观察者，当被观察者发生变化时，调用观察者的方法通知观察者完成更新

    （8）中介模式

    中介模式和观察者模式类似，但是中介模式处理的是同级之间的关系。

【JS】

  1. 行，那咱们问一些基础把，你觉得进程和线程有什么区别？

  2. 多线程有什么好处？

  4. 你了解浏览器缓存吗？

    答：强缓存和协商缓存

  5. 那这2个有什么区别？

    答：强缓存是通过设置请求头的expires和cache-control，想服务器请求会先看缓存，如果强缓存没有，就走协商缓存，如果协商缓存有，就返回304，如果没有，服务器返回新的资源

  6. 强缓存需要向后台请求吗？

    答：强缓存如果没有命中的话，就会走到协商缓存，如果命中的话，就直接从缓存返回资源呀

    【补充】强缓存表示在缓存期间不需要请求

  7. 向js、css这些静态资源存储在哪里？

    答：存储在service worker、内存，硬盘中

    【补充】内存缓存，主要存储一些小文件，读取速度非常快，具有时效性，当关闭Tab，缓存也释放了；硬盘缓存，主要存储一些大文件，一般比较大的css和js，读取速度慢

  10. 基础类型存在堆还是栈？

    答：基础类型存在堆，引用类型存在栈里（老师说这个你可能需要再看看）

    【补充】

    • 基础类型

      （1）空间确定，保存在栈中（2）保存和复制的都是值 （3）检测用typeof

    • 引用类型

      （1）空间不固定，保存在堆中，引用变量存储在栈中的是指向堆中的数组或者对象的地址（2）保存和复制都是指向对象的一个地址（3）检测用instanceof

    【补充】没有Promise之前,解决异步都是依赖回调

    （1）promise的诞生，主要是为了解决回调地狱的问题，在没有promise之前，js实现异步编程主要是通过嵌套回调函数实现的，嵌套繁多复杂的回调函数就会出现回调地狱。

    （7）异步的结果还可以通过回调函数，将结果做为（一般是第一个）参数传给回调函数

    （8）构造一个promise对象 return new Promise((reslove,reject)=>{}

  2. vue的computed和watcher的区别？

    答：computed的值没有发生变化，不会触发，watcher监听新旧值的变化（老师，那computed就相当于对一个计算的缓存）

    【补充】

    （1）computed支持缓存，只有数据发生改变时，才会发生改变

    （2）computed直接取缓存的值，methods每次都重新计算

    （3）不支持异步

    （4）computed适合对多个变量或者对象处理后返回一个结果值

    （5）computed监控自己定义的变量，然后可以直接在通过双向绑定在界面使用

    （6）watch不支持缓存，只要有值变化，就会触发

    （7）watch支持异步

    （8）watch监听的数据必须是data、props中的数据

  3. vue的组件通信的方式？

    答：父子组件props、emit，还有发布订阅模式

    【补充】

    （1）props、$emit

    父组件通过props向子组件传值，子组件通过$emit自定义事件向父组件传值

    （2）$emit、$on

    组件之间，不仅限父子组件

    通过 var Event = new Vue(); 做为事件的总线，Event.$emit(事件名,数据)、Event.$on(事件名，data=>{})

    （3）vuex

    vuex实现单向数据流，【vueComponent】 通过$store.disaptch('事件名'，data)触发【Actions】，然后通过$store.commit() 触发【mutation】，最后更新【state】

    （4）$attrs、$listeners

    $attrs 会包含父组件中没有被props的所有属性，通过v-bind=”$attrs”`可以传入内部组件

    $listeners 会包含父组件中所有v-on的事件，通过`v-on=”$listeners”可以传入内部组件

    （5）provide、inject

    祖先组件通过provide提供一个变量，然后子孙组件通过inject注入变量并使用

    （6）$parents、$children、ref

    ref用在DOM上，引用指向的就是DOM元素，ref用在子组件上，引用指向的就是组件实例
