  1. 原型链的了解

  2. css各种定位，每种定位相对于哪，每种定位的特点

    【回答】：

    • static:默认布局，没有定位，元素出现在正常的文本流中
  • relative:相对布局，相对于自己原来的位置
    • absolute:绝对布局，相对于某个上级元素（一般是父元素）并且有定位，有一个限制条件是上级元素不能是static定位，不然就会相对于网页的根元素html进行定位
    • fixed:固定布局，相对于浏览器视口
    • sticky:粘性布局，滚动到一定位置会固定

  1. flex布局的意义，再详细介绍。

    【回答】：

    （1）容器属性：flex-direction、flex-wrap、flex-flow（flex-direction+flex-wrap）、just-content、align-items、align-content（多轴）

    （2）项目属性：order、flex-grow、flex-shink、flex-basis、flex（flex-grow+flex-shink+flex-basis）、align-self

  4. 然后网络安全，详细介绍cookie，比较cookie sessionStorage localStorage 区别，怎么保护cookie不被泄露。

    【回答】：

    （1）尽量避免在cookies中存储敏感信息，比如e-mail、密码

    （2）如果浏览器端不需要对cookie操作，可以在Set-Cookies的末尾加上HttpOnly

    （3）cookie和本机的ip进行绑定

    （4）尽量使用post请求方式而不是get请求方式

  5. 之后介绍token，具体原理，怎么应用。

    【回答】：

    token和cookie的都是服务器端返回的，不同的是，token提升了服务器端的性能，服务器端不会再做保存操作；

    当客户端传递信息过来时，服务器端用相同的算法和密钥进行信息加密，然后比较前端传递的token和自己计算的token是否一致，如果一致，则返回相关资源，如果不一致，则返回错误代码。

  6. 详细介绍XSS，CSRF,怎么防范，它们之间的区别，

    【回答】：

    （1）XSS：跨站脚本攻击；攻击者将一段恶意代码注入到前端网页中，当用户访问时，会执行恶意代码，达到攻击的目的；前端可以将js的特殊字符替换成html编码；后端可以对提交的数据进行过滤。

    （2）CSRF：跨站请求伪造；攻击者盗用受害者的信息，去发送恶意请求；使用post不用get；在cookie中加入httponly；增强token

    （3）区别：CSRF需要登录，XSS不需要登录，XSS是攻击者向受害者的网页中加入恶意的js代码；CSRF是通过盗用受害者的身份，去完成恶意请求。

  12. Vue中的通信方式，传参方式，传参的底层原理（？？？没听太懂 没答出来）

    【回答】：

    vue组件之间的通信方式：

    1. 通过props向子组件传递数据：父 -> 子

    2. 通过事件向父组件发送消息：子 -> 父，使用$emit发送事件

    3. 父链 和 子索引：this.$parent 与 this.$children

    4. 依赖注入：provide 和 inject

    5. 子组件引用： ref与$refs

    6. 特性绑定：v-bind="$attrs" 和 v-on="$listeners"

    7. event bus

    vue的路由传参方式：

    直接携带式、name+params、path+query

  13. Vuex的设计原理，如果让你设计你会怎么设计，优缺点。

    【回答】：

    vuex涉及的相关知识：state、getters、mutations、actions、modules

    设计原理：

    （1）store通过vue的混入机制，将vuexinit混入vue的beforecreate之前，vueinit方法实现将store对象注册到vue组件实例中，并注册了store的引用属性$store。

    （2）state利用vue的data响应式，将store的state存入到vue的data中；getters利用vue的computed实现对数据的实时监听。
